電情3年生C課程ソフト実験2,3のレポート作成の手引き　2017.12 片山

・共通事項
  1. 実験において重要だった事柄は記述する。
     問題が生じて暫く停滞した場合、
       ・個人のひらめき
       ・最初の説明やテキストの内容がきっかけ
       ・教員側のてこ入れ
       ・他班のリーク情報 etc.
     の何が糸口となり、どのように解決したかを記述。

  2．特に問題が生じなかった場合でも、実験に際して事前に工夫し、
     理解を深めるためにどう思考整理したかを記述。

  3. テキストに載っている内容は引用に留める。丸ごと書き写しは
     無駄だし、内容を理解していないように見える。

  4. 一連の作業を1回実行した後に無限ループで停止させる構造の
     テストプログラムやタスクプログラム(思考実験を含む)は
     受理しない(＝採点対象とはならない可能性大)。

  
・ソフト実験2(テーマ1)のレポート
  1. テーマ1の内容に関する事
    a. システムコールに関する理解と,運用時に気を付けるべき事柄
    b. 現時点での動作を優先した実装と、テーマ2以降を視野に入れた上で
       考慮すべき事項を満たす実装

  2. テーマ2の思考実験について
    a. キューに関する条件は、ready, semaphore1, semaphore2の
       すべてのキューで満たすこと。
       更に、全てのタスクは繰り返しループ内に本体が記述されていること。
    b. ready, semaphore1, semaphore2の各キューの状態とは、
       キューに繋がれたタスクのリストである。
       (セマフォの場合はセマフォ状態値も含む)
    c. 各キューの状態変化には、タスク切り替えに関するイベントが伴うが、
       それらのイベントは思考実験開始以降の時間軸上にある。
    d. b., c. より、横軸にイベント内容と各キューの状態、縦軸にタスク切り替え関連
       イベントを発生順に並べれば、イベント発生に伴うキューの状態変化を
       表として記述できる。
    e. タスク数に関する制約は指定されていない。
    f. 思考タスクについてはタイミングに関する便宜性を重視し,
       実現可能性を保証する必要はない。
       しかし、いつどんなセマフォ命令を実行し、タスクのどの場所、
       どのタイミングでタイマ割込みが起きるかを恣意的に調整した
       タスクの構造を示すために、C言語に基づくプログラム記述を行い、
       イベントのタイミング調整のためのタスク内時間配分は、

       for (i=0;i<N;i++){ /* N: X秒間ループ、この待機ループの間にタイマ割込みが必ず生じる */ } 
       for (j=0;j<M;j++){ /* M: Y秒間ループ、この待機ループの間にタイマ割込みは生じない */ } 

       for (i=0;i<N;i++){ /* N: タスクループ１回につき、この待機ループで1回タイマ割込みが必ず生じる */ }
       for (j=0;j<M;j++){ /* M: タスクループ１回につき、この待機ループではタイマ割込みは生じない */ }

       の様にして行え。
       a. より、全てのタスクは一連の作業を繰り返し行う構造である。
       タスク内時間配分はタスクプログラム内のどこでイベントを
       発生させようとしているかの位置が判ればよく、実現可能性に
       係わる厳密な時間調整を行う必要はない。


・ソフト実験3(テーマ2,3)のレポート
  1. テーマ2について
    a. 分担作業なので、自分の担当した所は詳しく、
       更に自分の担当外の所は浅く触れることにより、
       システム全体に対する自分の担当部の位置づけを
       明確にする

  2. テーマ3について
    a. 共同開発部分については、共通事項に準じる。
    b. 個人開発部分について
      アプリケーションの説明と共に、以下の項目について必ず触れよ。
      ・マルチタスク性を生かした部分は何処か
      ・マルチタスクOSの特性を生かせる課題であるか事前に検討しているか
      特筆すべき点があれば、アピールポイントを明確にして記述。
